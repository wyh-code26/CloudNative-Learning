```text
The Kubernetes scheduler is a control plane component that assigns Pods to Nodes. 
The scheduler determines which Nodes are valid placements for each Pod in the 
scheduling queue according to constraints and available resources. The scheduler 
then ranks each valid Node and binds the Pod to a suitable Node. Multiple different 
schedulers may be used within a cluster; kube-scheduler is the reference implementation.
```

精读步骤：

1. 中英对照（5分钟）：
     Kubernetes调度器是一个控制平面组件，负责将Pod分配到节点上。调度器根据约束条件和可用资源，为调度队列中的每个Pod确定哪些节点是有效的放置位置。然后调度器对每个有效节点进行打分，并将Pod绑定到一个合适的节点。集群中可以使用多个不同的调度器；kube-scheduler 是参考实现。
2. 核心词汇：
     assign → 分配
     constraints → 约束条件
     rank → 打分/排序
     bind → 绑定
3. 底层映射思考：
     Scheduler的工作是“为任务找合适的机器”。这和你写C语言时，手动决定把哪个计算任务放在哪个线程/进程里执行是同一类问题——资源调度。
 面试锚点：
     Q: K8s Scheduler的基本工作流程是什么？ A: ①过滤（Filtering）：找出满足Pod资源需求和约束的节  点；②打分（Scoring）：对符合条件的节点打分排序；③绑定（Binding）：将Pod与得分最高的节点绑定。
