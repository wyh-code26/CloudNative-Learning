```text
The API server is a component of the Kubernetes control plane that exposes 
the Kubernetes API. The API server is the front end for the Kubernetes 
control plane.

The main implementation of a Kubernetes API server is kube-apiserver. 
kube-apiserver is designed to scale horizontally—that is, it scales by 
deploying more instances. You can run several instances of kube-apiserver 
and balance traffic between those instances.
```

精读步骤：

1. 中英对照（5分钟）：
   API服务器是Kubernetes控制平面的一个组件，它暴露了Kubernetes API。API服务器是Kubernetes控制平面的前端。
   Kubernetes API服务器的主要实现是kube-apiserver。kube-apiserver被设计为水平扩展——即通过部署更多实例来扩展。你可以运行多个kube-apiserver实例并在这些实例之间平衡流量。
2. 核心词汇提取（5分钟）：
     expose → 暴露
     front end → 前端（入口）
     scale horizontally → 水平扩展
3. 底层映射思考（15分钟）：
     API Server是K8s的“前台”，所有操作（kubectl命令、控制器查询）都先经过它。这和你之前部署的Nginx作为前端入口是一个模式。
     “水平扩展”意味着无状态，可以加多个实例。这和你在C语言里写过的无状态函数（每次调用不依赖上次结果）是同一思想。
     面试锚点：Q: kube-apiserver为什么可以水平扩展？ A: 因为它是无状态的，所有集群状态都存储在etcd中，多个apiserver实例共享同一个etcd即可。
