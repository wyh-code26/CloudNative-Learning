A Kubernetes cluster consists of a control plane plus a set of worker machines, called nodes, that run containerized applications. Every cluster needs at least one worker node in order to run Pods.

The worker node(s) host the Pods that are the components of the application workload. The control plane manages the worker nodes and the Pods in the cluster. In production environments, the control plane usually runs across multiple computers and a cluster usually runs multiple nodes, providing fault-tolerance and high availability.

1. 中英对照精读
一个 Kubernetes 集群由控制平面以及一组被称为节点的工作机器组成，这些节点运行容器化应用。每个集群至少需要一个工作节点，才能运行 Pod。
工作节点托管作为应用工作负载组件的 Pod。控制平面管理集群中的工作节点和 Pod。在生产环境中，控制平面通常运行在多台机器上，集群也通常运行多个节点，以此提供容错能力和高可用性。

2. 核心词汇提取
 
- control plane → 控制平面

- worker machines / nodes → 工作节点

- containerized applications → 容器化应用

- Pod → Pod（K8s 最小调度单元）

- application workload → 应用工作负载

- fault-tolerance → 容错

- high availability → 高可用
 
3. 底层映射思考
整个 K8s 集群采用控制平面 + 工作节点的分离架构：

- 控制平面：负责全局管理、调度、决策

- 工作节点：负责真正运行容器、承载业务 Pod
生产环境多节点、多控制平面部署，就是为了避免单点故障，实现集群高可用。

4. 面试锚点
Q：简述 Kubernetes 集群的基本组成及其作用。
A：K8s 集群由控制平面和工作节点组成。控制平面负责管理集群、调度 Pod；工作节点负责运行容器化应用的 Pod。生产环境会部署多控制平面和多节点，以实现容错与高可用。
