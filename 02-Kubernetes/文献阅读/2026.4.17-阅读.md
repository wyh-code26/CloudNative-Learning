```txet
Kubernetes is a portable, extensible, open source platform for managing 
containerized workloads and services, that facilitates both declarative 
configuration and automation.
```

1. 中文对照：
     Kubernetes是一个可移植的、可扩展的开源平台，用于管理容器化的工作负载和服务，
     它同时促进了声明式配置和自动化。
2. 核心词提取：
     portable 可移植的
     declarative configuration 声明式配置
     automation 自动化
3. 底层映射思考
     portable（可移植）：容器+K8s让应用不依赖特定服务器/环境，在哪都能跑。
     declarative configuration（声明式配置）：只告诉K8s想要什么状态（比如跑3个副本），不用写一步步命令，它自己去达到目标。
     automation（自动化）：扩缩容、自愈、重启、部署全自动，不用人一直盯着。
    面试锚点：
     Q：K8s 的核心特点是什么？
     A：可移植、可扩展、开源；支持声明式配置，高度自动化，统一管理容器化应用。



```text
A Kubernetes cluster consists of a set of worker machines, called nodes, 
that run containerized applications. Every cluster has at least one worker node.

The worker node(s) host the Pods that are the components of the application workload. 
The control plane manages the worker nodes and the Pods in the cluster. 
In production environments, the control plane usually runs across multiple computers 
and a cluster usually runs multiple nodes, providing fault-tolerance and high availability.
```

1. 中英对照精读：
      一个Kubernetes集群由一组称为节点的工作机器组成，这些节点运行容器化应用。每个集群至少有一个工作节点。
     工作节点托管着作为应用工作负载组件的Pod。控制平面管理集群中的工作节点和Pod。在生产环境中，控制平面通常跨多台计算机运行，集群通常运行多个节点，以提供容错和高可用性。
2. 核心词汇提取：
     worker node → 工作节点
     control plane → 控制平面
     fault-tolerance → 容错
     high availability → 高可用
3. 底层映射思考：
     “控制平面”和“工作节点”的分离，和之前配的 Master-Worker架构 是一回事。Master负责调度决策，Worker负责干活。
    面试锚点：
      Q: K8s集群由哪些核心组件构成？
      A: 控制平面组件（API Server、etcd、Scheduler、Controller Manager）和工作节点组件（kubelet、kube-proxy、容器运行时）。控制平面负责全局决策，工作节点负责运行Pod。
